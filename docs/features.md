# 機能一覧

## プロジェクト管理

「宣伝したい何か」をプロジェクトとして登録する。

プロジェクトは複数作成できる。OSSツール、技術記事、インディーゲーム、新しく開いた店のInstagramアカウント——名前をつけて自由に管理する。プロジェクトごとにダッシュボードが生成され、それぞれの宣伝活動を独立して追跡できる。

## 投稿URLの登録

プロジェクトに紐づけて、宣伝投稿のURLを登録する。URLを貼り付けるだけでよく、どのプラットフォームの投稿かは自動的に判別される。

ひとつのプロジェクトに対して、複数の投稿URLを登録できる。「Zennで記事を書いて、Redditでも宣伝した」という状況をそのまま管理できる。

## 自動スクレイピング

以下のプラットフォームは、URLを登録するだけでエンゲージメント数を自動取得する。

| プラットフォーム | 取得できる数値 |
|---|---|
| GitHub | Star数、Fork数、Issue数 |
| Zenn | いいね数、コメント数、ブックマーク数 |
| Qiita | いいね数、コメント数、ストック数 |
| Note | スキ数、コメント数 |
| Reddit | upvote数、コメント数 |

毎日1回、最新の数字が自動で保存される。ユーザーが何もしなくても、日々の記録が積み上がっていく。

## 手動入力

以下のプラットフォームは、各社の認証や有料APIが必要なため、自動取得に対応していない。数値を手入力することで記録できる。

- X（旧Twitter）
- Instagram
- YouTube
- TikTok
- Facebook
- Threads

手動であっても、入力した数値は時系列で保存され、グラフとして確認できる。

## 時系列グラフ

プロジェクトごとに、エンゲージメント数の推移をグラフで確認できる。

「あの記事を書いた翌週に Star が増えた」「Reddit 投稿の直後にいいねが伸びた」——そういった気づきが、グラフを見ることで得られる。数字の単点ではなく、流れとして把握できることがこの機能の価値である。

## 野球テーマのメッセージ

プロモーションの結果が振るわないとき、あるいは何もデータがないとき——そういう場面で、野球をテーマにした励ましのメッセージが表示される。

> 「三振を恐れるな、バットを振れ」
> 「全打席ホームランの選手はいない」
> 「さぁ、次の打席に立て！」

数字に落ち込みすぎず、次の発信へ向かうための後押しをする。結果を恐れずオープンに晒し続けることが大事だというメッセージが、サービス全体のトーンに流れている。

## 公開ダッシュボード（開発中）

自分のダッシュボードを公開URLとして共有できる機能を準備している。

「こんな感じで宣伝しています」と透明に見せることも、数字をそのままオープンに晒すことも、両方選べる。公開するかどうかはプロジェクトごとに設定できる予定である。
